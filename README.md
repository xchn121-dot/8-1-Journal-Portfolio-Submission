# 8-1-Journal-Portfolio-Submission
Reflection

Artemis Financial is a financial services company that wanted to improve the security of its software application used to transfer sensitive financial information. The company needed a secure communication solution that would protect customer data and reduce potential security risks. My task was to assess vulnerabilities in the application and implement secure coding practices to improve its overall security.

When reviewing the client's software, I was able to identify vulnerabilities through dependency analysis and security testing. I learned the importance of secure coding because it helps protect sensitive information, prevents unauthorized access, and reduces the likelihood of security breaches. Software security adds value to a company by protecting customer trust, maintaining compliance requirements, and preventing costly security incidents.

One of the most helpful parts of the vulnerability assessment was using automated tools to identify security risks and outdated dependencies. Understanding the results and determining the appropriate mitigation strategies was sometimes challenging, but it provided valuable experience with real-world security analysis.

To increase layers of security, I implemented secure communication using SSL/TLS certificates and addressed identified vulnerabilities through dependency management and code review practices. In the future, I would continue using vulnerability scanning tools, dependency-check utilities, and secure coding standards to assess risks and select appropriate mitigation techniques.

To ensure the application remained functional and secure, I tested the software after making changes and reviewed dependency reports to verify vulnerabilities had been resolved. After refactoring the code, I reran security scans and confirmed that no new vulnerabilities had been introduced.

Resources and tools that were helpful throughout this project included Maven, OWASP Dependency-Check, Java keytool, vulnerability reports, and secure coding best practices. These tools will continue to be useful in future software development and security-related projects.

For future employers, I could demonstrate my completed vulnerability assessment report and secure software practices report as examples of my ability to identify security risks, implement mitigation strategies, and apply secure coding principles within a software development project.
